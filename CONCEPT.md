# Creator — Концепция

## Главная идея

Визуальный редактор сайтов где дизайн = код с первого пиксела.

Под капотом не Figma-формат — а чистый HTML/CSS/JSON. Дизайнер рисует в знакомом интерфейсе, но рисует сразу веб-объекты. Никакой трансляции "макет → вёрстка".

**Проблема которую решаем:**

Сейчас один сайт живёт в трёх местах:
```
Figma (слои/вектор) → Webflow (классы/блоки) → Код (DOM/CSS)
```
На каждом переходе — потеря, ручная работа, несоответствие. Одна кнопка описана три раза тремя разными способами.

**Наш принцип:** Canvas = DOM-дерево. Рисуешь — это уже и есть код.

---

## Два режима

### Bird's eye
- Бесконечный холст со всеми артбордами проекта
- Видишь весь проект с высоты птичьего полёта как в Figma
- Можно зумить, двигать, видеть как страницы соотносятся
- Артборды в этом режиме не редактируются — только навигация
- Двойной клик на артборд → переход в Page mode

### Page mode
- Провалился внутрь артборда
- Это просто страница сайта — контент в потоке
- Панель слоёв, панель свойств, инструменты
- Escape → вернуться в Bird's eye

---

## Модель позиционирования

**Холст = DOM-дерево.** Никакого свободного размещения элементов по координатам XY без родителя.

### Flow-элемент (дефолт)
- Живёт в потоке документа
- position: static / relative
- Управляется flex/grid родителя
- Порядок определяется местом в дереве

### Pinned-элемент
- position: absolute
- Обязательно внутри конкретного контейнера (родитель = position: relative)
- Явная привязка к краю родителя (top / right / bottom / left)
- Понятное поведение при ресайзе
- Переключение в панели свойств: `В потоке / Закреплён`
- При выборе "Закреплён" — схема привязки (9 точек) к краям родителя

**Что запрещено:** absolute без родителя, элементы на холсте без контейнера, Figma-style "раскидать прямоугольники где попало".

## MVP (без AI)

Доказать главную идею: дизайн и код — одно и то же.

**Canvas**
- Добавить элемент (div, текст, картинка, секция)
- Drag внутрь контейнера (с подсветкой куда встанет)
- Ресайз за края
- Выделение кликом
- Вложенность

**Панель слоёв**
- Дерево элементов
- Drag для изменения порядка и вложенности
- Переименовать, удалить, дублировать

**Панель свойств**
- Размеры (width, height)
- Отступы (padding, margin, gap)
- Flex/Grid настройки
- Цвет фона, граница, скругление
- Типографика
- Переключатель В потоке / Закреплён + привязка

**Экспорт**
- Скопировать как HTML + CSS
- Скачать index.html

## Отличия от существующих инструментов

| | Figma | Webflow | Creator |
|---|---|---|---|
| Под капотом | SVG/вектор | Проприетарный формат | HTML/CSS/JSON |
| Позиционирование | Свободное XY | Поток + absolute | Поток + absolute с привязкой |
| Вид проекта | Все страницы на холсте | Только одна страница | Bird's eye + Page mode |
| Выход | Макет | Сайт на хостинге Webflow | Чистый HTML/CSS |
| Для кого | Дизайнер | Дизайнер/разработчик | Дизайнер который мыслит вебом |

---

## UI/UX

Визуально и по раскладке интерфейса — один в один как Webflow. Те же панели, та же структура, та же логика расположения элементов. Дизайнер который видел Webflow — сразу ориентируется.

**Единственное отличие:** только белая тема. Никакого тёмного интерфейса.

## Стек

- React + TypeScript
- Zustand — стейт дерева элементов
- @dnd-kit — drag & drop внутри дерева
- Vite — сборка

## Связь с другими инструментами проекта

- `design-inspector` — уже умеет читать и редактировать CSS в браузере, знания переиспользуем в панели свойств
- `figma2vibe` — умеет извлекать дизайн из Figma, в будущем может быть точкой импорта в creator
